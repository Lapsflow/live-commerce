# ONEWMS-FMS API 통합 계획서

## 1. 개요

### 목적
라이브 커머스 플랫폼과 ONEWMS-FMS(Fulfillment Management System) API를 연동하여 주문, 재고, 배송을 자동화합니다.

### 배경
- 현재 수동으로 처리되는 주문/재고/배송 관리를 자동화 필요
- ONEWMS API를 통한 실시간 재고 동기화 및 배송 추적
- 라이브 방송 종료 후 대량 주문의 효율적 처리

### 범위
- ✅ ONEWMS API 클라이언트 라이브러리 구현 (완료)
- 🔄 라이브 커머스 주문 시스템과 ONEWMS 연동
- 🔄 재고 실시간 동기화
- 🔄 배송 상태 자동 업데이트
- 🔄 관리자 대시보드에 ONEWMS 정보 표시

## 2. 비즈니스 요구사항

### 2.1 주문 관리 자동화
- 라이브 방송 종료 시 모든 주문을 ONEWMS로 자동 전송
- 주문 생성 시 실시간 재고 확인
- 재고 부족 시 알림 및 주문 보류 처리

### 2.2 재고 동기화
- 상품 재고를 ONEWMS와 실시간 동기화
- 주기적 재고 확인 (5분마다)
- 재고 부족 시 관리자 알림

### 2.3 배송 추적
- ONEWMS 배송 상태를 자동으로 플랫폼에 반영
- 송장번호 자동 업데이트
- 고객에게 배송 상태 알림

### 2.4 관리자 기능
- ONEWMS 연동 상태 모니터링
- 수동 재고 동기화 버튼
- 주문 재전송 기능

## 3. 기술 요구사항

### 3.1 API 클라이언트
- ✅ TypeScript 기반 클라이언트 라이브러리 (lib/onewms/)
- ✅ 모든 ONEWMS API 엔드포인트 지원
- ✅ 에러 핸들링 및 재시도 로직
- ✅ 타입 안전성 보장

### 3.2 데이터베이스
- ONEWMS 주문 매핑 테이블 추가
- 재고 동기화 히스토리 저장
- 배송 상태 로그 기록

### 3.3 백그라운드 작업
- Cron Job: 재고 동기화 (5분마다)
- Cron Job: 배송 상태 업데이트 (10분마다)
- Queue: 대량 주문 처리

### 3.4 보안
- ONEWMS API 키 환경 변수 관리
- API 호출 로그 기록
- 실패 시 재시도 메커니즘
