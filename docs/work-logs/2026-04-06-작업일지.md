### 1. Dashboard "Failed to fetch dashboard stats" 에러 수정 완료 ✅

**문제 상황**
- 프로덕션 환경(https://live-commerce-opal.vercel.app/dashboard)에서 로그인 후 Dashboard 접근 시 "Failed to fetch dashboard stats" 에러 발생
- 사용자가 통계 데이터를 볼 수 없는 상태

**해결 과정**

#### 1단계: NextAuth API Route 누락 수정
- **문제**: `app/api/auth/[...nextauth]/route.ts` 파일 완전 누락
- **증상**: 로그인 시도 시 `/api/auth/error` 404 에러
- **해결**: NextAuth handlers export하는 route 파일 생성
```typescript
import { handlers } from "@/lib/auth";
export const { GET, POST } = handlers;
```

#### 2단계: Vercel 환경변수 손상 복구
- **문제**: DATABASE_URL에 중복 prefix와 `\n` 문자 포함
  ```
  DATABASE_URL="DATABASE_URL=postgresql://...&channel_binding=require\n"
  ```
- **해결**: Vercel 환경변수 4개 모두 정상값으로 업데이트
  - DATABASE_URL
  - DIRECT_URL
  - AUTH_URL
  - AUTH_SECRET

#### 3단계: 로그인 폼 필드명 불일치 수정
- **문제**: 로그인 폼에서 `id` 필드 전송, 하지만 auth는 `email` 필드 기대
- **파일**: `app/(auth)/login/page.tsx`
- **해결**:
```typescript
// Before
const [id, setId] = useState("");
signIn("credentials", { id, password, redirect: false });

// After
const [email, setEmail] = useState("");
signIn("credentials", { email, password, redirect: false });
```

#### 4단계: Session Role 전파 문제 해결 (핵심 수정)
- **근본 원인**: Session 객체에 `role` 필드가 전달되지 않음
- **에러 경로**: Dashboard API → `getRoleBasedFilter()` → "알 수 없는 역할" 에러

**수정 파일들**:

1. **`lib/auth.ts`** - JWT/Session 콜백에 디버그 로깅 추가
```typescript
jwt({ token, user }) {
  if (user) {
    token.userId = user.id;
    token.role = user.role;
    token.adminId = user.adminId;
  }
  console.log("[JWT Callback] token.role:", token.role);
  return token;
},
session({ session, token }) {
  if (session.user) {
    session.user.userId = token.userId as string;
    session.user.role = token.role as string;
    session.user.adminId = token.adminId as string | undefined;
  }
  console.log("[Session Callback] session.user.role:", session.user?.role);
  return session;
}
```

2. **`lib/api/role-filter.ts`** - 방어적 에러 핸들링 추가
```typescript
if (!session?.user) {
  console.error("[Role Filter] No session or user");
  throw new Error("인증되지 않은 사용자");
}

if (!user.role) {
  console.error("[Role Filter] Missing role in session. User:", user);
  throw new Error("권한 정보가 없습니다. 다시 로그인해주세요.");
}
```

3. **`app/api/stats/dashboard/route.ts`** - 에러 핸들링 개선
```typescript
console.log("[Dashboard API] Session before role filter:", {
  hasSession: !!session,
  hasUser: !!session?.user,
  role: session?.user?.role,
  userId: session?.user?.userId,
});

// 권한 에러를 401로 명확히 반환
if (err.message?.includes("권한") || err.message?.includes("역할") || err.message?.includes("인증")) {
  return NextResponse.json(
    { error: err.message || "권한이 없습니다. 다시 로그인해주세요." },
    { status: 401 }
  );
}
```

4. **`lib/api/middleware.ts`** - TypeScript 타입 캐스팅 수정
```typescript
// Double casting으로 타입 호환성 해결
const authUser = toAuthUser(session.user as unknown as Record<string, unknown>);
```

5. **`types/next-auth.d.ts`** - 삭제
- 이유: `lib/auth.ts`에 이미 type extension 존재, 중복으로 인한 컴파일 에러 발생

**배포 상태**
- ✅ TypeScript 빌드 성공
- ✅ Vercel 프로덕션 배포 완료
- ⏳ 사용자 테스트 대기 중

## 📝 주요 학습 내용

1. **NextAuth v5 Session Flow**
   - authorize callback → JWT callback → session callback
   - 각 단계에서 데이터 전파가 명시적으로 이루어져야 함

2. **TypeScript Module Declaration**
   - 중복 타입 선언 주의 (lib/auth.ts vs types/next-auth.d.ts)
   - NextAuth 타입 확장은 한 곳에서만 수행

3. **Vercel 환경변수 관리**
   - 환경변수 값에 줄바꿈 문자 포함 시 연결 실패
   - 중복 prefix 주의

4. **Role-Based Access Control**
   - Session에 role 정보가 누락되면 전체 권한 시스템 작동 불가
   - 방어적 에러 핸들링으로 명확한 에러 메시지 제공 필요
